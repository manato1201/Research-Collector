// 01 · Overview

何を解決するシステムか

技術トレンドの変化が速く、分散した情報源を手動でキャッチアップするには限界がある。 本システムは収集・整理・蓄積を完全自動化し、NotebookLMをAI外部脳として活用することで トークン消費ゼロで大量の技術資料を検索・活用できる環境を実現する。

6 収集ソース種別
150 週間収集件数（上限）
3 年間運用可能（NotebookLM Plus）
0 AI検索時のトークン消費
📰
毎日自動収集
RSS経由でZenn・Qiita・Unity/UE・CEDECの新着記事を毎朝6時に自動収集
📄
論文収集
arXiv・Semantic ScholarからRAG/LLM/DCC学習関連論文を週2回（月・木）収集
🔁
重複チェック
SHA256ハッシュで収集済みURLを管理。GitHub Actionsをまたいでartifactで永続化
📚
週次ノートブック
カテゴリ別に週次ノートブックを自動作成。300件上限（Plus）を超えない設計
📋
週次レポート生成
収集内容からカテゴリ別日本語まとめレポートをNotebookLM CLIで自動生成
🔐
認証自動更新
Windowsタスクスケジューラで毎日AM5:30に認証更新。GitHub CLIでSecret自動更新

// 05 · Schedule

自動実行スケジュール

毎日 AM 5:30
NotebookLM 認証自動更新
Windows タスクスケジューラ → refresh_auth.ps1 → GitHub Secret 自動更新
毎日 AM 6:00
Daily Research Collect
GitHub Actions (Ubuntu) → RSS収集 → 重複チェック → NotebookLM追加

Weekly Research Digest
GitHub Actions → 当週ノートブックからカテゴリ別日本語レポート自動生成

Weekly Auth Check
